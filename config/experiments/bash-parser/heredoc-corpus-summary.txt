## Expected Parse Structure

Key fields in :expect:
- :heredoc-delimiter - The end marker (e.g., "EOF")
- :heredoc-content - The actual content between delimiters
- :heredoc-quoted - Boolean: true for <<'EOF', false for <<EOF
- :heredoc-indented - Boolean: true for <<-EOF (strips tabs)
- :variable-expansions - List of $VAR found in content
- :command-substitutions - List of $(cmd) found in content
- :redirects - File redirects after heredoc
- :pipeline - Boolean: heredoc output piped to another command

## Key Heredoc Patterns Covered

1. Delimiter Quoting Styles:
   - <<EOF (unquoted - allows expansion)
   - <<'EOF' (single-quoted - literal, no expansion)
   - <<"EOF" (double-quoted - allows expansion, same as unquoted)

2. Special Heredoc Features:
   - <<-EOF (indented heredoc - strips leading tabs)
   - Empty heredoc (delimiter immediately follows)
   - Custom delimiters (END, MARKER, SQL, PYTHON, etc.)

3. Integration with Other Features:
   - Inside command substitution $(cat <<'EOF')
   - With command substitution in content
   - With pipes (cat <<EOF | grep)
   - With file redirects (cat <<EOF > file)
   - In chained commands (&&)

4. Real-World Use Cases:
   - Git commit messages (multi-line with co-author)
   - SQL/Python scripts fed to interpreters
   - Markdown/documentation generation
   - Configuration file generation
