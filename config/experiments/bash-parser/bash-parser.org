* Introduction

Parse bash command strings using tree-sitter to extract semantic structure.

This is the main loader module that coordinates all bash parser subsystems.
The parser has been split into logical components for better maintainability:

- **bash-parser-core** - Core parsing infrastructure (pipelines, chains, commands)
- **bash-parser-glob** - Glob pattern matching without filesystem access
- **bash-parser-security** - Security validation with rule matching
- **bash-parser-semantics** - Command semantics database
- **bash-parser-variables** - Variable context management
- **bash-parser-file-ops** - File operations extraction
- **bash-parser-extensions** - Parser extensions (indirect ops, command injection)

** Capabilities

The parser extracts:
- Command names and subcommands
- Flags and positional arguments
- File operations with security semantics
- Dangerous pattern detection
- Command structure analysis (pipelines, chains, simple commands)

The parser handles multi-command constructs (pipelines and chains) by extracting
all commands and validating each one individually.

* Load Sub-Modules

Load all parser subsystems in dependency order.

#+begin_src emacs-lisp
;; Add current directory to load-path for sub-modules
(add-to-list 'load-path (file-name-directory (or load-file-name buffer-file-name)))

;; Protocol module (forward declarations - no dependencies)
(require 'bash-parser-protocol)

;; Core modules (no dependencies on other parser modules)
(require 'bash-parser-glob)
(require 'bash-parser-semantics)
(require 'bash-parser-variables)

;; Security module (depends on glob)
(require 'bash-parser-security)

;; Core parsing (depends on security - for dangerous pattern detection)
(require 'bash-parser-core)

;; File operations (depends on core, semantics, variables)
(require 'bash-parser-file-ops)

;; Recursive analysis (depends on file-ops)
(require 'bash-parser-recursive)

;; Extensions (depends on file-ops)
(require 'bash-parser-extensions)
#+end_src
