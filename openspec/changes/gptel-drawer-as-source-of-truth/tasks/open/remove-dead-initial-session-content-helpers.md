## Why

End-of-cycle Architect finding `arch-cycle-1779477564-6` (advisory, class `dead-branch`). Both helpers emit the OLD session.org body shape (`:PROPERTIES:` drawer + `#+begin_user` block, no `* System Prompt`, no `* Chat`). Cycle-7's `emit-system-prompt-and-chat-headings-at-creation` rewired `jf/gptel--create-session-core` to use the new `jf/gptel--session-headings-block` / `jf/gptel--initial-session-body` pair, leaving these two helpers production-dead.

The cycle's minimal action was to update the docstring on `jf/gptel--initial-session-content` ("This helper is retained for direct callers and helper-level tests") — but there are no direct callers, and the helper-level tests pin the OLD shape, which is misleading for a future reader looking for "what a fresh session.org looks like" in the test suite. Grouping the deletion with the parallel persistent-agent helper avoids two single-helper cleanup tasks for the same class.
