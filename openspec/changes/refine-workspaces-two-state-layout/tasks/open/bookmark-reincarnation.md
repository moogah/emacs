## Implementation steps

1. **Define `workspace-buffer` struct.** In `data-model.org`, add a new section between *Layout primitives* and *Layout group primitives*:

   ```elisp
   (cl-defstruct workspace-buffer
     (bookmark nil)
     (filename nil)
     (name nil)
     (narrowed-p nil)
     (indirect-p nil)
     (local-variables nil)  ; reserved; always nil in v2
     (etc nil))
   ```

   Provide accessor functions following the existing `workspace--layout-*` naming convention if any of the auto-generated `workspace-buffer-*` accessors collide with something useful. The autogenerated ones should be fine.

2. **Add the leaf walker** in `layouts.org` as a new section *Window-state leaf walker*. Model on `activities--mapcar-window-state-leafs` and `activities--window-serialized` (`runtime/straight/repos/activities.el/activities.el:527-711`). The walker is a `pcase`-based recursive descent:

   - `(leaf . attrs)` form: apply `func` to the leaf attrs.
   - Atom: return as-is.
   - Cons of key and atom: return as-is.
   - List: `mapcar #'walk` over its members.

   Two variants:
   - `workspace--window-state-walk-leaves` — pure walker that takes a callback.
   - `workspace--window-state-serialize` / `workspace--window-state-deserialize` — apply the walker with a leaf-translation function that adds (resp. consumes) the `workspace-buffer` entry on each leaf's `parameters` map.

   **Edge case**: a one-window frame's top-level form is `(leaf . attrs)` directly, not wrapped in a multi-window list. Detect with `(cl-position 'leaf state)`; if non-nil, treat the form as a single leaf and translate just its attrs. Catalog pattern 2 has the exact recipe.

3. **Rewrite `workspace--capture-frameset`** to:
   1. Call `window-state-get` with the extended `window-persistent-parameters` (already done from pattern 4 commit).
   2. Run the leaf walker with a translation function that calls `(workspace--serialize-buffer (get-buffer NAME))` for each leaf's saved buffer-or-buffer-name, and stores the resulting `workspace-buffer` struct in the leaf's `parameters` map under key `workspace-buffer`.

4. **Write `workspace--serialize-buffer`** as a method (or plain `defun`) that takes a live buffer and returns a `workspace-buffer`:

   ```elisp
   (defun workspace--serialize-buffer (buffer)
     (when (buffer-live-p buffer)
       (with-current-buffer buffer
         (make-workspace-buffer
          :bookmark (condition-case err
                        (bookmark-make-record)
                      (error nil))   ; non-bookmarkable buffers
          :filename (buffer-file-name buffer)
          :name (buffer-name buffer)
          :narrowed-p (buffer-narrowed-p)
          :indirect-p (and (buffer-base-buffer buffer) t)))))
   ```

5. **Rewrite `workspace--restore-frameset`** to:
   1. `copy-tree` the saved window-state form so the in-memory registry is not mutated by step 2.
   2. Run the leaf walker with a translation function that calls `workspace--deserialize-buffer` on each leaf's `parameters.workspace-buffer` entry; the result replaces the leaf's `buffer` slot.
   3. Defer the actual `window-state-put` via `(run-at-time nil nil ...)` — this avoids racing against `bookmark--jump-via`'s buffer-display call.
   4. The deferred timer closure captures a *generation* counter (see step 6); on fire, it no-ops if a newer restore has been queued since.

6. **Write `workspace--deserialize-buffer`** implementing the four-step fallback chain:

   ```elisp
   (defun workspace--deserialize-buffer (ws-buffer)
     (or
      ;; 1. Bookmark restore
      (and (workspace-buffer-bookmark ws-buffer)
           (condition-case-unless-debug err
               (workspace--bookmark-buffer ws-buffer)
             (error nil)))
      ;; 2. Filename fallback
      (and (workspace-buffer-filename ws-buffer)
           (ignore-errors (find-file-noselect (workspace-buffer-filename ws-buffer))))
      ;; 3. Name fallback
      (and (workspace-buffer-name ws-buffer)
           (get-buffer (workspace-buffer-name ws-buffer)))
      ;; 4. Error buffer
      (workspace--error-buffer ws-buffer)))
   ```

7. **Write `workspace--bookmark-buffer`** with the `help-mode` natively-compiled-subr workaround (Emacs `bug#56643`). The reference implementation is `activities--bookmark-buffer` at `activities.el:810-853`. The shape is:
   - Use a `with-temp-buffer` to detect whether `bookmark-jump` actually moved off the temp buffer.
   - `condition-case` wraps the `bookmark-jump` call so a `read` error on a compiled subr falls through cleanly.

8. **Write `workspace--error-buffer`** that returns a live, named buffer (e.g. `*workspace-restore: <orig-name>*`) containing a one-paragraph explanation of which fallback step failed. This buffer is what shows in the window slot when reincarnation fails entirely. Model on `activities--error-buffer` at `activities.el:855-862`.

9. **Add the generation counter** in `persistence.org` (or `layouts.org`, wherever the restore entry-point lives):

   ```elisp
   (defvar workspace--restore-generation 0)
   ```

   Increment on every entry to `workspace--apply-saved-layout`; capture into the deferred timer closure; compare on fire.

10. **Remove the explicit `find-file-noselect` pre-load** from `workspace--apply-saved-layout`. The leaf walker handles it.

11. **Specs**:
    - In `data-model-spec.el`, add a describe-block `workspace-buffer struct` with cases: construct/access by accessor; nil bookmark accepted; narrowed/indirect flags round-trip via prin1/read.
    - In `layouts-spec.el`, extend the existing window-state describe block with: capture embeds `workspace-buffer` in each leaf; one-window-frame form is handled; the extended `window-persistent-parameters` (from the earlier commit) still apply.
    - In `buffer-reincarnation-spec.el` (new file), cover:
      - Bookmark path succeeds → returns the bookmark's buffer.
      - Bookmark path errors → falls through to filename.
      - Filename path succeeds for file-backed; nil for non-file.
      - Name path succeeds when a live buffer of that name exists; nil otherwise.
      - All four fail → returns the error buffer (asserts it's live and named).
      - `bug#56643` shape: feed in a bookmark record whose `read` raises; assert fall-through to filename.
      - Deferred restore: mock `run-at-time` and assert the closure is queued, not immediate.
      - Generation guard: queue two restores; only the second's closure body runs.
      - `magit-status` reincarnation: mock `bookmark-handle-bookmark` to return a buffer named `magit: ...`; assert leaf buffer becomes that buffer.
